How to Use VX6 — Installation and Setup

This guide covers the complete process of installing VX6, initialising a node, establishing a connection to a peer, and routing application traffic through the tunnel. Estimated setup time for an experienced user on a pre-configured IPv6 network: under five minutes.

1 — Prerequisites #

1.1 — IPv6 connectivity #

VX6 requires a publicly routable IPv6 address. To verify that your machine has one:

# Linux / macOS
$ curl -6 ifconfig.me
2001:db8:cafe::1   # a public IPv6 address

If your machine only shows a link-local address (fe80::) or no IPv6 address at all, contact your ISP to enable IPv6, or use a tunnel broker such as Hurricane Electric to obtain a routable address. VX6 does not function over NAT64 on the initiating side; the node itself must be publicly reachable.

1.2 — Supported platforms #

Platform Minimum version Notes
Linux Kernel 5.4 (2019) Most distributions. Requires CAP_NET_ADMIN for tunnel interface.
macOS 12 (Monterey) System Integrity Protection must permit network extensions.
Windows 10/11 with WSL2 Run vx6 inside WSL2. Native Windows support planned for Phase 2.

1.3 — Firewall requirements #

Allow inbound and outbound traffic on port 9600 for both TCP and UDP. VX6 prefers QUIC (UDP) for performance; TCP is used automatically if UDP is unavailable.

# Linux (ufw)
$ ufw allow 9600/tcp
$ ufw allow 9600/udp

2 — Installation #

The installer script downloads the latest signed binary, verifies its SHA256 checksum, and adds vx6 to your system PATH.

# Linux / macOS
$ curl -sSL https://vx6.io/install.sh | bash
 
# Windows — run PowerShell as Administrator
PS> iwr https://vx6.io/install.ps1 | iex
 
✔  VX6 v1.0.0 installed at /usr/local/bin/vx6
Note: Always verify the installer script before piping it to your shell. The SHA256 checksum for each release is published at vx6.io/checksums.

3 — Node initialisation #

Run vx6 init once after installation. This command generates an Ed25519 keypair, derives your VX6 IPv6 address from the public key, and writes a configuration file to ~/.vx6/config.toml.

$ vx6 init
 
Generated keypair  ~/.vx6/identity.ed25519
Derived IPv6       fd47:6b8::cafe:babe
Node ID            vx6:Ed25519:a3f9c2...b7d2
Config             ~/.vx6/config.toml
 
✔  Node initialised. Back up ~/.vx6/identity.ed25519 — it cannot be recovered.
Warning: Back up ~/.vx6/identity.ed25519 immediately. Your VX6 IPv6 address is derived from this key. If you lose it, your address cannot be recovered and peers will not be able to reach you at the same address.

4 — Connecting to a peer #

A peer can be dialled by its raw IPv6 address or by its Node ID. Both methods result in the same direct connection — no routing servers are consulted during dialling.

# Dial by IPv6 address (with port)
$ vx6 connect [2001:db8::dead:beef]:9600
 
# Dial by Node ID (resolves to IPv6 via DHT in Phase 2; direct in Phase 1)
$ vx6 connect vx6:Ed25519:f7a2...de01
 
Resolved          2001:db8::dead:beef:9600
Handshake         Noise_XX — mutual auth OK
Latency           3 ms
Session key       ChaCha20-Poly1305 established
 
✔  Tunnel active. All traffic is end-to-end encrypted.

5 — Routing traffic #

VX6 exposes two interfaces for routing application traffic:

  • SOCKS5 proxy — available on 127.0.0.1:1080. Configure any SOCKS5-capable application to point at this address.
  • NAT64 interface — a virtual network interface that transparently intercepts IPv4 traffic and routes it through the VX6 tunnel. Applications require no modification.
# Start the SOCKS5 proxy
$ vx6 proxy --socks5 127.0.0.1:1080
✔  SOCKS5 proxy listening on 127.0.0.1:1080
 
# Test with curl
$ curl --proxy socks5h://127.0.0.1:1080 https://example.com
HTTP/2 200
 
# Enable the NAT64 transparent interface
$ sudo vx6 nat64 up
✔  NAT64 interface vx6nat0 active

6 — Verifying status #

$ vx6 status

Peers        2 connected
Latency avg  3 ms
Uptime       00:14:38

7 — Configuration reference #

The configuration file is located at ~/.vx6/config.toml. The following keys are recognised:

KeyDefaultDescription
port9600Listening port for inbound connections (TCP and UDP).
identity~/.vx6/identity.ed25519Path to the Ed25519 private key file.
transport"quic"Preferred transport. Options: "quic", "tcp", "auto".
socks5_bind"127.0.0.1:1080"Bind address for the SOCKS5 proxy. Set to "" to disable.
nat64falseEnable the NAT64 transparent interface on startup.
log_level"info"Verbosity. Options: "error", "warn", "info", "debug", "trace".
