# Contributing to VX6 Web

Thanks for taking the time to contribute.

## How to contribute

1. Fork the repository and create a feature branch from `main`.
2. Install dependencies and run local checks.
3. Make focused changes with clear commits.
4. Open a pull request with a clear summary.

## Coding standards

- Keep components small and reusable.
- Use descriptive names for variables and functions.
- Keep styling changes minimal and aligned with existing theme variables.
- Add concise comments only when behavior is not obvious.

## Pull request process

- Provide a short summary of the change and why it is needed.
- Link related issues or discussions.
- Note any UI changes and include screenshots when relevant.
- Confirm lint and build checks pass.

## Issue reporting format

Please include:

- Title: concise description
- Context: page or feature area
- Steps to reproduce: numbered list
- Expected result
- Actual result
- Screenshots or logs (optional)
