## About Section

VX6 is a service-first peer-to-peer network for real applications.
It lets one machine expose a local service to another machine without turning that service into a public internet endpoint.
You can share SSH, web apps, databases, internal tools, file transfer, and hidden services by name, invite, or direct node path.
The result is simple: localhost stays local, while access becomes global to the people you choose.

## Short Product Description

VX6 is a decentralized service-sharing network for teams, operators, builders, and private infrastructure.
It connects people to services, not just to servers.
The current release supports Linux on the `main` branch and Windows on the `Windows-compatible` branch, with the same current protocol and feature behavior.

## Feature Blocks

### Localhost to Localhost

Turn `127.0.0.1:22`, `127.0.0.1:8080`, or `127.0.0.1:5432` into a service another node can reach from its own localhost.

### Public and Private Services

Use names like `alice.ssh` or `team.api` for public services, and use private per-user catalogs for services that should not appear in public discovery.

### Hidden Services

Publish a service by invite-backed alias and route it through intro, guard, and rendezvous paths without exposing the real service endpoint.

### DHT-Backed Lookup

VX6 uses signed records, multi-source lookup checks, bounded replication, and local cache to find nodes and services without making every node store everything.

### Linux and Windows Support

Linux is the current source-of-truth branch and Windows support is available now through the `Windows-compatible` branch.

### Real App Ready

SSH, dashboards, APIs, databases, internal tools, admin panels, chat backends, and team workflows all fit naturally.
