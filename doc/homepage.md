## About Section

VX6 is an IPv6-first peer-to-peer network for real applications.  
It lets one machine expose a local service to another machine without turning that service into a public internet endpoint.  
You can share SSH, web apps, databases, internal tools, file transfer, and private services by name, alias, or direct IPv6 address.  
The result is simple: localhost stays local, while access becomes global to the people you choose.

## Short Product Description

VX6 is a decentralized service-sharing network for teams, operators, builders, and private infrastructure. It connects people to services, not just to servers. It is built for direct use, relay paths, hidden services, and peer-to-peer applications on modern IPv6 networks, and it can grow outward from any known live node.

## Feature Blocks

### Localhost to Localhost

Turn `127.0.0.1:22`, `127.0.0.1:8080`, or `127.0.0.1:5432` into a service another node can reach from its own localhost.

### Named Services

Use names like `alice.ssh` or `team.api` instead of remembering raw IPs.

### Direct IPv6 Mode

If two people already know each other's IPv6 addresses, they can connect directly with no extra network layer.

### Hidden Services

Publish a service by alias and route it through relay nodes without exposing the service endpoint.

### Peer Discovery

Start from any known live node, learn the network through peers, and let signed records and DHT lookups keep the mesh moving.

### Hidden Routing

Reach private services by alias through intro nodes, guards, rendezvous points, and relay paths.

### Real App Ready

SSH, dashboards, APIs, databases, internal tools, admin panels, chat, and team workflows all fit naturally.
